1 Introducción

1.1 Alcance del proyecto

El alcance del sistema "InfoSiles" afectará a los siguientes aspectos de la organización HERMANOS SILES S.L :

  • Gestión de pedidos de los clientes mediante una plataforma online donde podrán comprar los productos y elegir si desean que se les envíen a domicilio o recogerlo en tienda.

  • Gestón de empleados mediante una lista con los datos de los empleados de la empresa y sus respectivos contratos.

  • Gestión de proveedores mediante una lista con los datos de los proveedores y un historial de productos pedidos a estos.

  • Gestión del inventario de productos mediante un catálogo de productos y su stock en el almacén. Además con la posibilidad de renovación de stock con la automatización de pedidos.

1.3 Objetivos del proyecto

Los principales objetivos que se esperan alcanzar con el nuevo sistema son:

  • Mejora de visibilidad del sistema.

  • Aumentar el alcance del negocio

  • Aumento de las ventas.

  • Automatización de los pedidos a los proveedores.

  • Mejora de gestión del inventario.

  • Mejora de gestión de pedidos.

  • Mejora en la venta de productos.

  • Informatizar el negocio de forma general.

2.2 Descripción de la arquitectura del sistema

En nuestro sistema, el usuario accede a la capa de presentación para poder manipular el sistema, esto se consigue gracias a que la capa de presentación está conectada con la capa de lógica de negocio que es la que se encarga de implementar las operaciones. A su vez, la capa de negocio se comunica con la capa de persistencia que es la encargada de leer, guardar y eliminar los datos de la base de datos.

En caso de que el usuario sea un empleado, podría acceder a una API externa con el fin de enviar correos a proveedores de forma semiautomatizada.

6.2 Descripción de la interfaz de servicios del sistema

En nuestro sistema contemplamos 5 servicios.

Servicio de Clientes permite a los clientes de la empresa registrarse en el sistema, consultar su perfil y actualizar la información asociada a éste, realizar pedidos y consultar los que ya ha realizado previamente. Cuenta con los siguientes métodos:

  • RegistrarCliente() es un método que permite añadir a cada cliente sus datos personales, tales como DNI, nombre, apellidos... y registrarlos en nuestra base de datos.

  • ActualizarCliente() permite a los clientes actualizar sus datos, recibiendo como parámetros los del anterior método.

  • ConsultarMiPerfil() ayudará a los clientes de forma individual a consultar sus datos personales, estos se mostrarán en una vista de perfil personal. Allí encontraremos también los alquileres, pedidos realizados y pedidos pendientes.

  • RealizarPedidoCliente() registrará un pedido desde el servicio cliente. Este método hará uso además del servicio de pedidos (RegistrarPedido(),ConfirmarPedido(),PagarPedido()). Recibirá una lista de productos y los datos personales de clientes. Únicamente se invocará cuando un cliente desee tramitar un pedido desde pedidos pendientes en Mi Perfil. También recibirá una cuba y la fecha del pedido, permitiendo así consultar la disponibilidad de esta.

Servicio de Pedidos permite realizar un nuevo pedido, siendo posible añadir productos al pedido, modificarlo, confirmarlo y pagarlo.
En este caso se usará SendGrid para enviar pedidos a los proveedores, a través del método EnviarPedidoAProveedor(). Cuenta con los siguientes métodos:

  • RegistrarPedido() este método recibirá como parámetros la lista de productos, el usuario del cliente, la fecha de creación y el estado de pago. Simplemente registrará un pedido a X cliente, será usado principalmente en la tienda en físico (aunque en la online también nos sirva). Registrará el pedido en la base de datos.

  • AñadirProducto() recibirá el pedido actual y el producto a añadir, se encargará de añadirle el producto nuevo a este pedido.

  • ActualizarPedido() este método recibirá los nuevos parámetros del pedido y se encargará de actualizar su estado.

  • ConfirmarPedido() a través de esta invocación, el empleado o cliente podrá actualizar el estado de un pedido. Por defecto recibirá dicho estado y el pedido en sí. El cliente lo hará de forma pasiva, una vez se reciba su pago. Este estado se obtiene mediante otro método que obtiene el estado.

  • PagarPedido() a través de este método, el propio servicio ejecutará el pago del pedido. Deberá recibir el pago asociado, el cliente y el pedido en sí. Este se llamará cuando haga falta actualizar el estado del pedido.

  • GetEstadoPedido() con este método, el sistema podrá calcular si un pedido ha sido pagado o no. Este recibirá el pedido, a través del cual obtendrá las facturas asociadas, de ellas se comprobarán las fechas de emisión y pago a fin de obtener dicho estado.

  • EnviarPedidoAProveedor() este método recibirá como parámetro el proveedor y el producto. Al ser invocado, estará enlazado al método send de la API de sendgrid que enviará un correo al proveedor.

Servicio de Administradores permite al administrador registrar nuevos empleados, así como sus contratos y modificarlos. Además pueden registrar nuevos proveedores, actualizarlos y gestionar las facturas, generándolas tanto para los clientes como para los proveedores. Cuenta con los siguientes métodos:

  • RegistrarEmpleado() con la invocación de este método, el administrador podrá insertar un empleado en la base de datos, una vez obtenidos los datos asociados a este.

  • ConsultarEmpleados() a través de este método, el usuario administrador podrá consultar todos los empleados registrados. Este método no requiere parámetros.

  • ActualizarEmpleado() invocando este método mediante un empleado asociado actualizado, cambiará sus atributos en la base de datos.

  • BuscarEmpleadoPorTipo() recibiendo un tipo empleado en este método, se buscará una lista filtrada por dicho tipo.

  • AñadirContrato() con este método el administrador podrá registrar un nuevo contrato en la BD, recibiendo por parámetros los campos principales de esta entidad.

  • ConsultarContratos() invocando este método desde el usuario administrador se nos mostrarán todos los contratos actuales de la BD. No recibe parámetro.

  • ActualizarContrato() el administrador también podrá modificar los datos de un contrato a través de este método. Este recibirá el nuevo contrato modificado.

  • RegistrarProveedor() este es el método asociado a los formularios de proveedores, que permitirá introducirlos en la base de datos y recibe como parámetros todos los datos de estos.

  • ConsultarProveedores() a través de este método, el usuario administrador podrá consultar todos los proveedores registrados. Este método no requiere parámetros.

  • ActualizarProveedor() el administrador también podrá modificar los datos de un proveedor a través de este método. Este recibirá el nuevo proveedor modificado.

  • RegistrarFactura() este es el método asociado a los formularios de facturas, que permitirá introducirlas en la base de datos y recibe como parámetros todos los datos de estas, además del pedido asociado a ella.

  • ConsultarFacturasIngresos() este método forma parte de la generación de informes. Recibirá com parámetros las fechas que se desean estudiar y calculará los ingresos del negocio a través de facturas de clientes.

  • ConsultarFacturasGastos() este método forma parte de la generación de informes. Recibirá com parámetros las fechas que se desean estudiar y calculará los gastos del negocio a través de facturas de proveedores y contratos.

  • ConsultarContratoPorEmpleado() con los nombres de un empleado, el sistema buscará los contratos asignados a dicho empleado.

Servicio de Productos permite, tanto a los empleados como al administrador del sistema, registrar un nuevo producto, además de consultarlos y buscarlos por su nombre. Cuenta con los siguientes métodos:

  • RegistrarProducto() los empleados y administradores podrán usar este método para registrar un producto en base de datos. Por parámetro recibiríamos el producto en sí.

  • ActualizarProducto() los empleados y administradores podrán usar este método para actualizar un producto de la base de datos. Por parámetro recibiríamos el producto actualizado.

  • ConsultarProductos() los empleados y administradores podrán usar este método para obtener el listado de productos en stock. No recibiría ningún parámetro.

  • BuscarProductoPorNombre() a través de este método, el empleado podrá buscar un producto por su nombre. Este método devolverá el producto fruto de la búsqueda.

  • AñadirProductoACarrito() este método recibirá por parámetro un producto y el id del cliente. A través de él, se le asignará al carrito el producto dado.

Servicio de Empleados permite a los empleados de la empresa consultar su perfil y su contrato, consultar los clientes, realizar pedidos a los proveedores, consultar los pedidos tanto de clientes como los realizados a proveedores, buscar los pedidos asociados a un cliente o a un proveedor y consultar aquellos pedidos que estén activos. Cuenta con los siguientes métodos:

  • ConsultaMiPerfil() a través del id de empleado asociado, este método le mostrará a los trabajadores su perfil, conteniendo este los datos y atributos asociados a la entidad.

  • ConsultaMiContrato() un empleado podrá acceder a su contrato mediante la invocación a este método, que recibirá el id asociado y devolverá los datos principales del contrato.

  • ConsultarClientes() un empleado también podrá acceder al listado de clientes mediante esta llamada, por tanto, devolverá un listado de aquellos que estén registrados en la base de datos.

  • RealizarPedidoAProveedor() un empleado podrá emitir un correo a un proveedor dadas la dirección de email de proveedor y la de usuario. Este invocará al método EnviarPedidoAProveedor(). Este método recibirá el proveedor y el producto en cuestión.

  • ConsultarPedidosDeClientes() - ConsultarClientes() un empleado también podrá acceder al listado de pedidos de clientes mediante esta llamada, por tanto, devolverá un listado de los pedidos emitidos por estos.

  • ConsultarPedidosAProveedores() un empleado podrá acceder al listado de pedidos a proveedores mediante esta invocación, por tanto, devolverá un listado de los pedidos emitidos a estos.

  • BuscarPedidosPorCliente() este método recibirá como parámetro el usuario asociado al cliente y mostrará todos los pedidos relacionados con este.

  • BuscarPedidosPorProveedor() este método recibirá como parámetro el nombre asociado al proveedor y mostrará todos los pedidos relacionados con este.

  • ConsultarPedidosActivos() la invocación de este método creará un listado de pedidos pendientes de entrega.

  • AñadirCamion() este método recibirá los datos de la entidad camión para registrarlos en la base de datos.

  • ModificarCamion() este método será invocado usando como parámetro el nuevo camión modificado y lo actualizará en la base de datos.

  • AñadirIncidenciaCliente() el empleado podrá añadir una incidencia de cliente usando como parámetros el cliente asociado y la incidencia validada.
